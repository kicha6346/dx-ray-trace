IMPORTANT NOTICE FOR HACKATHON JUDGES

Thank you for evaluating DX-Ray Trace! This tool requires Node.js 18+ and takes approximately 60 seconds to install and run. Please follow the steps below. The full interactive experience works best in a proper terminal — not an IDE output panel.

To see the most impressive features, we recommend pointing the tool at any real project on your machine that has environment variables or npm dependencies. A demo project is included in the examples/sample-project folder in the repo.

1
Install globally: npm install -g dx-ray-trace — or clone the repo and run npm install && npm run build && npm link
2
Navigate to the example project: cd examples/sample-project
3
Run the interactive menu: dx — then select 🔍 Scan Environment
4
After the scan, use arrow keys to select an issue and watch the inline auto-fix launch instantly
5
Return to the menu and try 🧪 Simulate Fresh Install to see the Onboarding Difficulty Rating
📋

Installation — Option A (npm global)

Install globally in one command

Terminal

Verify installation

Terminal
dx --version

Launch the interactive menu

Terminal
dx
🔧

Installation — Option B (from source)

Clone the repository

Terminal
git clone https://github.com/kicha6346/dx-ray-trace.git cd dx-ray-trace

Install dependencies and build

Terminal
npm install npm run build

Link globally so 'dx' command works everywhere

Terminal
npm link

Done! Now run

Terminal
dx

🗂️

All Commands

dx
Launch the full interactive menu with arrow-key navigation
dx init [dir]
Auto-generate a dx-spec.yaml by introspecting your environment
dx scan [dir]
Run a 10-category scan and generate the Compatibility Matrix + DX Health Score
dx fix [dir]
Auto-fix detected issues with guided prompts or in turbo auto mode
dx simulate [dir]
Simulate a fresh developer install and get an Onboarding Difficulty Rating
dx export [dir]
Export your environment as a portable YAML snapshot for peer sharing
dx compare <file>
Compare your machine against a teammate's exported snapshot
✨

What It Scans

🟢
Runtime Versions
Validates Node.js and Python versions match the project specification exactly
📦
Dependencies
Checks npm and pip packages are installed and node_modules is present
🔑
Environment Variables
Detects missing keys, placeholder values, and .env vs .env.example drift
🛡️
Security Scanner
Finds hardcoded localhost URLs, Stripe keys, and AWS tokens in your source code
🐳
Docker Images
Verifies required Docker images are pulled and locally available
🔌
Port Conflicts
Detects blocked ports that would prevent your app from starting
🗂️
Git Hygiene
Ensures .gitignore protects node_modules, .env, and sensitive patterns
📝
Docs Freshness
Checks README existence and flags stale documentation
🏗️
Project Structure
Validates critical directories and entry point files exist
